Entendiendo almidon

Almidon fue desarrollado par desarrollar y hospedar sitios web, tiene entonces centralización, agilidad, seguridad y desempeño como principales metas.

Permance instalado sólo una vez, centralizando así el código, facilitando actualizaciones y mejroas. Cada sitio web en el servidor hace uso de esta misma instalación.

Almidon tiene una administracion generalmente en /admin o /secure que debe de preferencia accederse usando SSL. La administracion permite revisar, modificar, agregar datos a la base de datos de manera sencilla a traves de formularios.

Tiene tambien herramientas para desarrollo web rapido, ambas partes de almidon acceden a la base de datos con los datos especificados en classes/config.php, generalmente un usuario con acceso total desde la administracion y un usuario de solo lectura desde el sitio web publico.

En ambos casos se accede a la base de datos usando la estructura de datos especificada en tables.class.php.

Esquema:

/-------------------------------ALMIDON-------------------------------\
| www.example.com:                                                    |
| [Administracion] <--> [tables.class] <---ADMIN---> [Base de datos]  |
|      [Sitio web] <--> [tables.class] <-READ_ONLY-> [Base de datos]  |
|                                                                     |
| www.example.org:                                                    |
| [Administracion] <--> [tables.class] <---ADMIN---> [Base de datos]  |
|      [Sitio web] <--> [tables.class] <-READ_ONLY-> [Base de datos]  |
|                                                                     |
| www.example.net:                                                    |
| [Administracion] <--> [tables.class] <---ADMIN---> [Base de datos]  |
|      [Sitio web] <--> [tables.class] <-READ_ONLY-> [Base de datos]  |
